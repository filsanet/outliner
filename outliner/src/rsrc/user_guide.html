Installation and Setup
Muy Importante: if you encounter any problems following these installation and setup instructions, andor your installation just outandout fails, please contact us ASAP via http://JavaOutlineEditor.org/feedback
---------
Prepare your system
Windows
Ensure that your system has an appropriate Java 2 Runtime Environment (JRE) installed
Determine if you have a JRE, andor a Java 2 SDK (JSDK), and if so, which version(s) of each
Give the Start:Settings:Control Panel command
The Control Panel window appears
Start up the Add/Remove Programs applet
Look for Java 2 Runtime... and Java 2 SDK... entries in the list of installed software
Note the version numbers on those items
Analyze that information
Two goals:
a JRE that is version 1.3.1 or higher
if a JSDK is present, its version should match the JRE version
If you have only a JRE, and it is version 1.3.1 or above, you're okay
If you have both a JSDK and a JRE, the version numbers should match
If they don't, you'll need to uninstall them, and then install a 1.3.1 or higher JSDK, which will give you a matching JRE
If you have two or more entries for either the Runtime Environment or the SDK, you need to uninstall earlier versions, keeping only the highest-numbered version of each
If your JRE andor SDK is less than version 1.3.1, you need to uninstall those, and then install a 1.3.1 or higher version
Download sites
Java 2 SDK 1.3.1_02
http://java.sun.com/j2se/1.3/download-windows.html
JRE 1.3.1_02
http://java.sun.com/j2se/1.3/jre/download-windows.html
Uninstall and Install as necessary
To uninstall:
Give the Start:Settings:Control Panel command
The Control Panel window appears
Start up the Add/Remove Programs applet
Look for the Java 2 Runtime... or Java 2 SDK... entry you wish to remove in the list of installed software
Click the Add/Remove button
To install:
Get the download from one of the download sites noted above.
Execute it.
Ensure that there your system has sufficient environment space
Environment space is where DOS/Windows stores string constants
It can default to a very small amount, 256 bytes
To increase environment space generally, add a line like this to the file c:\config.sys
shell=c:\windows\command.com /e2048 /p
In this example, Windows is installed on the c drive in a directory named windows, and we are setting the environment to 2048 bytes
You can skip this general increase and just increase the environment space of the run.bat file that starts up JOE
Macintosh OS/X
Linux/Unix/Solaris
Download the latest JOE
Go to http://sourceforge.net/projects/outliner
Halfway down the page, in a section labeled "File Releases", you'll find the latest release of JOE that's appropriate for your operating system
Windows: outliner-win
Mac OS X: outliner-macosx
Linux/Unix/Solaris: outliner-win for now <more specific file forthcoming>
Install the download
Windows
Extract the Outliner directory and its contents from the download file.
If you haven't increased environment space generally, you can instead increase it for the Outliner directory's run.bat file
Find run.bat in the Outliner directory
Right-click on it and give the Properties menu command
Select the Memory tab in the run.bat Properties dialog
Set the Initial Environment widget to a minimum of 2048
Click OK
Mac OS X
Linux/Unix/Solaris
Create some startup shortcuts
Windows
Create shortcuts to the Outliner directory's run.bat file.
Mac OS X
Linux/Unix/Solaris
Start JOE up
Windows
Double-click one of your startup shortcuts
It'll take JOE 6-15 seconds to start up on a 900 mhz Windows system. The first startup of the day is the slowest.
Mac OS X
Linux/Unix/Solaris
Set JOE application preferences
Give the Edit:Application Preferences menu command
Go through the preference panels, and set items as desired.
That's it. Give a File:New/Open/Import command, and start outlining !

XML-RPC
About XML-RPC
XML-RPC lets you make remote procedure calls over HTTP.
JOE uses Hannes Wallnoefer's Helma XML-RPC package. Thanks Hannes.
http://xmlrpc.helma.org
Further info
Portal: : http://xmlrpc.org
Spec: http://xmlrpc.com/spec
Creating an XML-RPC Macro
Give the Script:Macro Manager... menu command. The Macro Manager window appears.
Select XMLRPC from the new macro category dropdown widget.
Click the New button. A New Macro dialog box appears.
The dialog box prompts you for the following:
Macro Name
This is the name that shows up in the macro popup.
I typically name my XML-RPC macros something like "XMLRPC Servername". [MD]
Server
The server name or ip-address of the XML-RPC server this macro will talk to.
This defaults to "127.0.0.1" the loopback address for accessing a server running locally.
Port
The port the server is running on.
This defaults to "8088" for no really good reason other than eights kind of go with HTTP.
Do Replacement
If "yes" then the response from the server will replace the current selection.
If "no" then the response will be shown in a dialog only.
This defaults to "no".
Click the Create button and you're done.
Using an XML-RPC Macro
Select an outline node.
The selected node should contain text, or a sub-tree that looks something like the following:
<?xml version="1.0"?>
<methodCall>
<methodName>method_name</methodName>
<params>
<param><value>Some Value</value></param>
</params>
</methodCall>
Right-click the selected node to bring up the macro popup menu.
Select the macro you want to apply.
You may have to wait a while, depending on how long the operation takes.
The macro blocks the application, so the macro must complete before you can continue working.
The current implementation of the XML-RPC macro requires that the selected node contain a valid XML-RPC call.
At some point it would probably be good to create an XML-RPC macro that uses the selection as a parameter to a call stored in the macro itself, but that's not how things work currently.
Strategies
One strategy I use when I have a call that takes one dynamic argument, is to create a "text replacement" macro that constructs the XML-RPC call. I then call the XML-RPC macro. [MD]

Menu Commands
File
New
keystroke shortcut: Ctrl+N
Creates a new outline, and opens it up in a window for editing. The new outline has a name of the form Untitled <#>, where <#> is a number that tracks the number of new documents created during this JOE session. (Untitled 1, Untitled 2, Untitled 3, etc)
Open
keystroke shortcut: Ctrl+O
Opens an existing outline. The outline must be in a format that JOE can both read and write. JOE can currently read and write these formats:
OPML
Text (also known as Plaintext)
You can open files via:
The local file system
The PHP web protocol
<forthcoming> The HTTP web protocol
<forthcoming> The FTP web protocol
Note: the Import command lets you open an existing outline that's in a format that JOE can read, but not write.
Recent Files
---
Import
Opens an existing outline that's in a format that JOE can read, but not write. JOE can currently import these formats:
Brain Forest Palm pdb
Shadow Plan Palm pdb
Thought Manager Palm pdb
You can import files via:
The local file system
The PHP web protocol
<forthcoming> The HTTP web protocol
<forthcoming> The FTP web protocol
Note: Since JOE cannot write Import formats, the Save command is grayed out when an imported file is topmost. You must use Save As to save an imported file, since you must specify the format to write it to.

OPML
outline
An outline is a collection of nodes that are organized in two dimensions: serial and hierarchic. The topmost level of an outline's hierarchic organization is called level 0. Nodes that are hierarchically subjunctive to a given node are called subnodes , or descendants, of that node. Subnodes one level down from a given node are called child nodes, and the given node is their parent. Nodes that share a parent node are called sibling nodes. The topmost set of sibling nodes is said to be at level 0 of the outline. Nodes contain arbitrary sorts of information, some of which is commonly displayed to the user.
